# 💾 Backup

Учебный проект по созданию и автоматизации резервного копирования данных.

Проект демонстрирует основные подходы к сохранению данных, созданию резервных копий и восстановлению информации в случае сбоев, ошибок пользователей или потери данных.

---

## 📋 Описание

Резервное копирование (Backup) — это процесс создания копий данных для последующего восстановления в случае повреждения, удаления или потери оригиналов.

В рамках проекта рассматриваются:

* создание резервных копий файлов;
* автоматизация бэкапов;
* работа с файловой системой;
* архивирование данных;
* восстановление информации;
* организация безопасного хранения резервных копий.

---

## 🎯 Цели проекта

## 📦 Создание резервной копии

Пример копирования файла:

```python
import shutil

shutil.copy(
    "source/data.txt",
    "backup/data.txt"
)
```

Копирование каталога:

```python
import shutil

shutil.copytree(
    "source",
    "backup"
)
```

---

## 🗜 Архивирование данных

Создание ZIP-архива:

```python
from zipfile import ZipFile

with ZipFile("backup.zip", "w") as archive:
    archive.write("data.txt")
```

## ♻️ Восстановление данных


## ⏰ Автоматизация резервного копирования

Добавление текущей даты в имя архива:

```python
from datetime import datetime

filename = (
    f"backup_"
    f"{datetime.now():%Y%m%d_%H%M%S}.zip"
)
```

---

## 🛡 Рекомендации по резервному копированию

Для надёжного хранения данных рекомендуется использовать правило **3-2-1**:

* 3 копии данных;
* 2 разных носителя хранения;
* 1 копия вне основного устройства.

Такой подход широко считается одной из базовых стратегий защиты данных.
